### L22：Mongo BSON DateTime 時區處理紀律
- `MongoDateTimeSerializer.Deserialize` (line 23): `DateTimeOffset.FromUnixTimeMilliseconds(ms).DateTime` 回傳 DateTime.Kind=Unspecified (UTC clock time、kind metadata 不帶)
- Mongo BSON Date 在資料庫內為 UTC: `"$date": "2026-04-28T06:35:51.775Z"`
- Golden Recipe 期望本地時區 (Asia/Taipei) + 無 Z 後綴: `"2026-04-28T14:35:51.775"` (06:35 + 8h = 14:35)
- 轉換樣板:`DateTime.SpecifyKind(Utc) → TimeZoneInfo.ConvertTimeFromUtc(TaipeiTz) → ToString("yyyy-MM-ddTHH:mm:ss.fff")`
- TaipeiTz resolver 須 3 段 fallback (`Asia/Taipei` → `Taipei Standard Time` → CustomTimeZone+8h)、避免 Windows / Linux ID 差異
- 既有 `DateTimeNoZConverter` 是 JsonConverter (序列化時觸發)、不適合在 Convert helper 階段直接呼叫;對 Public Model 型別為 `string` 的欄位、需在 Convert helper 內手動轉
