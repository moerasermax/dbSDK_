## 本 session 關鍵學習點

### L8：任務膠囊 VCP 必須主動驗證
- Engineer 在完成任務後，必須主動對照膠囊中的 VCP 驗收項目執行驗證
- 不需要等 PM 要求才做，這是 Engineer 的責任

### L9：程式碼重構要完整，不只改核心邏輯

- PM 提醒後才補上，下次要一次性完整修改

### L10：dump 命令需要輸出 JSON 格式
- `dump-sN` 命令的用途是輸出 JSON 供 PM 對比客戶 GoldenRecipe
- 與 `validate` 不同，validate 是跑驗證邏輯
- 需要在方法中加入 `verbose` 參數控制 JSON 輸出

### L11：檔案命名要有溯源前綴
- 新 Suite 類別命名格式：`{Phase}_SearchScenarioSuite.cs`
- 前綴 `P2_` 代表 Phase 2 (JSON Format Alignment)
- 方法上方要加 `[Sxx]` XML 註解標註原始 Sprint 編號

### L12：交接格式標準化
- 轉交給其他 AI 角色時使用格式：
  ```
  致 [角色]
  {請他做的內容}
  ---
  原因: [說明為什麼需要交接]
  ```
- 這是為了支持未來系統自我升級，確保任務正確傳遞
